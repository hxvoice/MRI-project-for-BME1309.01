cMRF workflow tips
==================

The real cMRF scanner workflow now uses one shared configuration source:

1. Edit `CMRF_DATA_ROOT` in `pipeline_config.py`, or set the environment
   variable `CMRF_DATA_ROOT` on the server.
2. Run the full workflow with:

   python main_cmrf.py

3. Override common parameters from the command line when needed:

   python main_cmrf.py --device cuda --n-iter 60 --lambda-llr 0.005 --matching-batch-size 2048

4. Single-step debugging is still available:

   python scripts/extract_cmrf_data.py
   python scripts/run_recon_cmrf.py
   python scripts/template_matching.py

Expected dataset layout:

CMRF_DATA_ROOT/
  cMRF_fa_705rep.txt
  scanner1/
    cMRF.h5
    processed/
      mrf_dictionary_data.npz
      mrf_kspace_noisy.npy
      traj_full_2d.npy
      csm.npy

The processed files are generated by `scripts/extract_cmrf_data.py`.
